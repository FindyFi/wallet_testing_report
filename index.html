About this report

These results were generated by Findy Wallet Tester, an open-source automated interoperability test suite against Pensiondemo issuers and verifiers. You can find more about them from the following links.

Status key
  • Pass — The full protocol flow completed successfully end-to-end.
  • Fail — The flow did not complete. The wallet or agent rejected the request, timed out, or returned an unexpected response.
  • Error — The test runner hit an unexpected exception before the flow could be evaluated. The result is inconclusive.
Reading the matrix

Rows are agents (issuer or verifier services); columns are wallet apps. A cell shows whether the wallet successfully completed a testing flow with an agent. The Issuance section covers credential offer and acceptance; Verification covers presentation requests and responses. A passing diagonal (same vendor on both sides) does not imply cross-vendor compatibility, meaning each cell is independent.

Questions or issues?

Results reflect a single automated run and may not capture all real-world scenarios. Passed results reflect fully working compatibility within testing limitations but failures can stem from configuration differences, transient network conditions, or test environment limitations rather than fundamental incompatibility. If a result looks incorrect or there are wallets or agents which should be added to testing suite, please rase an issue in the Findy Wallet Tester Github repository. We welcome feedback and contributions to the project.
